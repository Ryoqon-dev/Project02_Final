프로젝트명: 당신의 지역은 괜찮아요?
영문명: Is it safe around where you LIVE?

================================================================================
1. 프로젝트 개요
================================================================================

본 프로젝트는 경찰청에서 제공하는 공공데이터(범죄 발생 지역별 통계, 경찰관 1인당 담당 
인구수 현황)를 활용하여 사용자가 선택한 지역의 범죄 발생 현황과 치안 인프라 상태를 
콘솔 애플리케이션을 통해 직관적으로 확인할 수 있도록 하는 C 언어 기반 클라이언트-서버 
프로그램이다.

사용자는 프로그램 시작 시 로그인 또는 회원가입을 한 뒤, 지역 치안 분석 메뉴를 통해
선택한 지역(도/시 및 시/군/구)의 범죄 통계, 경찰 인프라, 실질 치안 위험도, 
맞춤형 안전 가이드라인을 조회할 수 있다.

핵심 가치:
- 복잡한 통계 데이터를 사용자 친화적인 "등급"과 "메시지"로 변환
- 단순 범죄 건수뿐만 아니라 경찰 인프라와 함께 고려한 "실질 위험도" 산출
- 범죄 유형별 맞춤형 안전수칙 제공으로 실질적 도움 제공


================================================================================
2. 개발 목적 및 배경
================================================================================

2.1 문제 인식
경찰청 공공데이터는 매우 유용한 정보를 담고 있지만, CSV 형식의 원본 데이터는:
- 숫자 중심으로만 구성
- 지역 간 비교를 위해 통계 계산 필요
- 일반인이 직관적으로 이해하기 어려움
- 데이터의 의미를 파악하기 위해 추가 분석 필요

예) "경기도 안산시의 범죄 건수가 500건이다"라는 정보만으로는 
    이 지역이 안전한지 위험한지 판단하기 어려움.

2.2 해결 방안
본 프로젝트는:
- 전국 평균 대비 비율 계산
- 범죄 발생도와 경찰 인프라를 결합한 실질 위험도 산출
- 숫자를 "안전/주의/위험" 등으로 등급화
- 범죄 유형별 맞춤형 안전수칙 제공

2.3 기대 효과
- 사용자가 자신의 거주지역 또는 관심 지역의 치안 상태를 빠르게 파악
- 데이터 기반의 객관적인 위험도 평가
- 범죄 유형별 예방 조치를 통한 실질적 안전 증대
- 공공데이터의 활용도 증대

4. 사용자 및 운영 환경
================================================================================

4.1 사용자 정의
   대상: 지역 치안 정보를 알고 싶어 하는 일반인
   - 아파트 이사 가능 지역의 치안 확인을 원하는 직장인
   - 자녀 교육지역의 안전도를 확인하려는 학부모
   - 새로운 도시로 이사 예정인 이주자
   - 특정 지역의 범죄 현황을 분석하려는 연구자/기자
   - 지역 치안 정보 제공 앱/웹의 백엔드 데이터 소스 활용자

4.2 운영 환경
   서버 운영자: 이 프로그램을 실행하는 기관/단체 관리자
   역할:
   - Server.exe를 먼저 실행하여 클라이언트의 접속을 대기
   - CrimeStats.csv, PoliceStats.csv 데이터 최신 유지
   - 종료 시 Server.exe 정상 종료

4.3 시스템 환경
   운영 체제: Windows (Windows 7 이상)
   네트워크: TCP/IP 로컬 네트워크 또는 인터넷
   하드웨어: 특별한 요구사항 없음 (표준 PC 사양)
   기억 장치: CrimeStats.csv와 PoliceStats.csv 로드 시 약 100MB 이내

4.4 실행 방식
   1단계: Server.exe 실행 (포트 9000 대기)
   2단계: Client.exe 실행 (127.0.0.1:9000으로 연결)
   3단계: 사용자 로그인/회원가입
   4단계: 지역 선택 및 통계 조회
   5단계: 결과 확인 후 로그아웃 또는 다른 지역 조회
   6단계: 프로그램 종료

================================================================================
[REQ-01] 지역별 범죄유형 Top 3 및 범죄 발생 통계 현황
================================================================================

요구사항 ID: REQ-01
우선순위: HIGH (필수 기능)
상태: 구현 완료

설명:
사용자가 도/시 단위 지역과 시/군/구 단위 지역을 번호로 선택하면, 서버는 
CrimeStats.csv 파일을 기반으로 선택된 지역의 모든 범죄 유형별 발생 건수를 
조회한다. 조회된 결과는 발생 건수 기준 내림차순으로 정렬하여 전체 목록을 
출력하고, 그중 가장 많이 발생한 범죄 유형 3가지를 Top 3으로 별도 표시한다.

입력:
- 도/시 번호 (1~17)
- 시/군/구 번호 (각 도/시별로 다름, 예: 충청남도의 1~14 범위)

처리:
1. 클라이언트에서 PKT_REQ_01_CRIME_TOP3 패킷 전송
2. 서버에서 해당 지역 인덱스 확인
3. CrimeNode 연결 리스트 순회하여 선택 지역의 범죄 건수 수집
4. CrimeRank 배열에 범죄명과 건수 저장
5. SortCrimeRanks() 함수로 내림차순 정렬
6. Top 3 범죄 유형 확인 및 강조

- 지능범죄 지역 → 지능범죄 가이드라인 검증
- 강력범죄 지역 → 강력범죄 가이드라인 검증
- 여러 범죄 혼합 → 우선순위대로 첫 번째 매칭 검증


================================================================================
7. 비기능 요구사항 (Non-Functional Requirements)
================================================================================

7.1 성능 (Performance)
- 서버 시작 시 CSV 파일 로드 시간: 5초 이내
- 클라이언트 요청에 대한 서버 응답 시간: 1초 이내
- 동시 클라이언트 처리: 최소 100명 이상 (이론상)
- 메모리 사용량: 약 50~100MB (CSV 데이터 로드 후)

7.2 신뢰성 (Reliability)
- 서버가 한 클라이언트의 오류로 인해 종료되지 않음
- 패킷 손상 감지 및 처리 (페이로드 길이 검증)
- 불완전한 데이터 수신 감지 및 재시도
- 클라이언트 연결 끊김 시 우아한 종료

7.3 보안 (Security)
- 사용자 비밀번호: 평문 저장 (실제 운영 환경에서는 암호화 권장)
- 패킷 페이로드 길이 검증으로 버퍼 오버플로우 방지
- 사용자 입력 최대 길이 제한 (NAME_SIZE, REGION_SIZE 등)
- SQL Injection: 해당 사항 없음 (CSV 파일 기반)

7.4 사용성 (Usability)
- 모든 출력이 한글로 표시 (UTF-8 코드페이지 사용)
- 직관적인 메뉴 구조 (번호 선택 방식)
- 결과 출력 후 엔터 대기로 사용자가 결과 확인 가능
- 오류 메시지가 명확하고 친절함

7.5 유지보수성 (Maintainability)
- 코드가 모듈화되어 있음 (함수 단위 기능 분리)
- 공통 헤더파일(Common.h)로 코드 중복 최소화
- 설정값(포트, 버퍼 크기 등)은 상수로 정의
- 주석과 함수명으로 의도가 명확함

7.6 확장성 (Scalability)
- 새로운 도/시 추가 시: ProvinceInfo 배열에 항목 추가
- 새로운 범죄 유형 추가: CrimeNode 리스트에 자동 로드
- 새로운 분석 기능 추가: 새 패킷 타입 정의 후 핸들러 작성
- 데이터 소스 변경: CSV 파싱 로직 수정으로 대응 가능

9. 데이터 검증 및 예외 처리 기준
================================================================================

9.1 로그인/회원가입 검증

   [회원가입 검증]
   1. ID 중복 확인
      - 기존 users.txt에서 해당 ID 검색
      - 존재하면 "이미 존재하는 아이디입니다"
   
   2. 비밀번호 검증
      - 길이 최소 1자 (권장 4자 이상)
      - 공백만 입력 금지
   
   3. 나이 검증
      - 정수 범위: 1~150
      - 범위 벗어나면 "유효한 나이를 입력하세요"
   
   4. 거주 지역 검증
      - 공백 입력 금지
      - 한글 및 공백 포함 가능

   [로그인 검증]
   1. ID 존재 여부 확인
   2. 비밀번호 일치 여부 확인
   3. 불일치 시 "아이디 또는 비밀번호가 올바르지 않습니다"

9.2 지역 선택 검증

   [도/시 번호 검증]
   - 범위: 1~17
   - 범위 벗어나면 "잘못된 도/시 번호입니다"

   [시/군/구 번호 검증]
   - 범위: 1 ~ (해당 도시의 시/군/구 개수)
   - 범위 벗어나면 "잘못된 시/군/구 번호입니다"

9.3 데이터 로드 검증

   [CSV 파일 로드]
   - 파일 존재 여부 확인
   - 파일 열기 실패 시 에러 메시지 + 프로그램 종료
   - 파일 파싱 에러 시 해당 행 건너뛰고 계속 로드

   [users.txt 로드]
   - 파일 없으면 자동 생성 (공파일)
   - 형식 오류인 행은 건너뛰고 계속 로드

9.4 네트워크 통신 검증

   [패킷 수신 검증]
   - 페이로드 길이가 양수인지 확인
   - 페이로드 길이 > 버퍼 크기이면 거부
   - 불완전한 수신 감지 시 연결 종료

   [소켓 오류 처리]
   - send/recv 반환값 확인
   - -1 반환 시 연결 종료
   - 각 클라이언트의 오류는 해당 스레드만 종료

11. 필수 기술 요구사항
================================================================================

[TECH-01] 소켓 프로그래밍
설명:
본 프로그램은 서버/클라이언트 구조로 구현되어야 한다.
Server.exe는 TCP 소켓을 열고 클라이언트 접속을 대기한다.
Client.exe는 서버에 접속하여 로그인, 회원가입, 지역 선택, 통계 조회 요청을 전송한다.

검증 기준:
- Server.exe 실행 후 포트 9000에서 대기한다.
- Client.exe가 서버에 접속하여 요청과 응답을 주고받는다.
- 서버 미실행 시 클라이언트는 연결 실패 메시지를 출력한다.

[TECH-02] 연결 리스트 구현 및 사용
설명:
프로그램 내부에서 단순 배열만 사용하는 것이 아니라 연결 리스트를 직접 구현하여 사용해야 한다.

적용 위치:
- UserNode: 사용자 계정 목록 관리
- CrimeNode: 범죄 통계 CSV 행 목록 관리
- PoliceNode: 경찰 통계 CSV 행 목록 관리
- ClientNode: 접속 중인 클라이언트 목록 관리

검증 기준:
- CSV와 사용자 파일을 읽은 데이터가 연결 리스트 노드로 저장된다.
- 서버는 연결 리스트를 순회하며 검색, 계산, 응답 생성을 수행한다.

[TECH-03] 데이터구조 과목 자료구조 1종 응용
설명:
배열 및 단순 연결 리스트를 제외한 자료구조를 1종 이상 응용해야 한다.
본 프로젝트는 최대 힙(Max Heap)을 구현하여 범죄 발생 건수 순위 계산에 사용한다.

적용 위치:
- CrimeMaxHeap 구조체
- HeapPush 함수
- HeapPop 함수
- SortCrimeRanks 함수

검증 기준:
- 선택 지역의 범죄 유형별 발생 건수를 최대 힙에 삽입한다.
- 발생 건수가 가장 큰 범죄부터 꺼내어 Top 3와 전체 순위를 만든다.

[TECH-04] 공공데이터 및 파일입출력
설명:
프로그램에 필요한 데이터는 공공데이터포털에서 다운로드한 CSV 파일을 사용해야 한다.
서버는 파일입출력을 통해 CrimeStats.csv와 PoliceStats.csv를 읽고 분석한다.
회원 정보는 users.txt 파일에 저장한다.

검증 기준:
- CrimeStats.csv 파일이 없으면 서버가 오류를 출력한다.
- PoliceStats.csv 파일이 없으면 서버가 오류를 출력한다.
- 회원가입 시 users.txt에 사용자 정보가 저장된다.

12. 사용자 피드백 반영 요구사항

[UI-01] 결과 요약 우선 출력
설명:
처음 테스트한 사용자는 통계 결과가 길게 출력되어 핵심 정보를 파악하기 어렵다고 피드백하였다.
따라서 범죄 통계 결과는 전체 목록보다 요약 정보를 먼저 출력해야 한다.

출력 순서:
1. 지역명
2. 총 범죄 발생 건수
3. 전국 평균
4. 평균 대비 비율
5. 범죄 안전 등급
6. Top 3 범죄 유형
7. 전체 범죄 유형별 발생 현황

[UI-02] 표 형태 출력
설명:
전체 범죄 목록은 단순 문장 나열이 아니라 순위, 범죄 유형, 발생 건수 열을 가진 표 형태로 출력해야 한다.

검증 기준:
- 각 범죄 유형이 줄 단위로 구분된다.
- 순위와 발생 건수가 한눈에 보인다.
- Top 3는 전체 목록과 별도 영역으로 표시된다.

[UI-03] 실질 위험도 화면 구조 개선
설명:
실질 치안 위험도 화면은 사용자가 계산 결과와 행동 지침을 쉽게 구분할 수 있어야 한다.

출력 순서:
1. 위험도 요약
2. 계산식
3. 경찰 인프라 통계 적용 단위 안내
4. 우선 행동 안내
5. 주요 범죄 유형별 맞춤 가이드
6. 공통 지침

13. 맞춤형 치안 가이드라인 상세 요구사항
================================================================================

[GUIDE-01] Top 3 범죄 유형별 개별 가이드라인 출력
설명:
기존처럼 하나의 범죄 유형만 기준으로 가이드라인을 출력하면 지역 특성이 충분히 반영되지 않는다.
따라서 선택 지역의 Top 3 범죄 유형 각각에 대해 개별 안전 수칙을 출력해야 한다.

[GUIDE-02] 범죄 유형별 가이드라인 매핑
설명:
서버는 범죄명에 포함된 키워드를 기준으로 다음과 같은 가이드라인을 매칭한다.

- 사기: 문자 링크, 기관 사칭 전화, 계좌 이체 요구를 공식 채널로 재확인
- 절도: 현관과 창문 잠금 확인, 차량과 자전거 귀중품 방치 금지
- 교통범죄: 야간 보행, 이면도로, 음주운전 의심 차량 주의
- 폭력범죄: 심야 유흥가와 좁은 골목 통행 자제, 시비 상황 회피
- 성범죄: 밝은 길 이용, 위치 공유, 안전귀가 서비스 활용
- 강력범죄: 직접 제압보다 112 신고와 주변 도움 요청 우선
- 경제범죄: 계약 증빙 보관, 비공식 투자 권유 주의
- 마약범죄: 출처 불명 약물, 음료, 택배 수령 주의
- 도박범죄: 불법 도박 사이트 링크와 고수익 홍보 차단
- 기타범죄: 반복 위험 장소 기록 및 지자체·경찰 신고 채널 활용

[GUIDE-03] 실질 치안 위험도 해석 보완
설명:
범죄 통계는 시/군/구 단위이고 경찰 인프라 통계는 도/시 단위이므로,
실질 치안 위험도 출력 시 선택 지역의 상위 도/시 경찰 인프라 값을 적용한다는 설명을 함께 제공해야 한다.

검증 기준:
- 위험도 화면에 계산식이 출력된다.
- 경찰 인프라 적용 단위 안내 문장이 출력된다.
- Top 3 범죄 유형 각각의 가이드라인이 출력된다.
